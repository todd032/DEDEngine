# Prototypes

This directory is for **fully separate prototypes** that should not disturb the main
`Cookie On The Roof X` product code.

## Classification Rules

- Keep each prototype in its own folder under `prototypes/<prototype-name>/`.
- Do not place prototype-only experiments directly under `src/`, `tests/`, or `android/`.
- Existing projects and references must remain untouched unless a task explicitly asks to integrate.
- Every prototype folder should include:
  - `README.md`: purpose, scope, and run/build notes
  - `STATUS.md`: current progress, risks, and next steps

## Active Separate Prototypes

- `roofless-prototype`: empty starter bucket for a new standalone concept.
