# Mesh Slicing 고정 컨텍스트 주입

본 문서는 Mesh Slicing 프로토타입 전반에 주입되어야 하는 **고정 컨텍스트(fixed context)** 를 정의한다. 아래 항목은 설계/구현/검증 단계에서 변경 없이 유지한다.

## 1) 문제 공간
- 목표는 입력 메시를 절단 평면으로 분할하고, 절단면(교차 윤곽)을 기반으로 후속 처리 가능한 구조를 만드는 것이다.
- 결과는 시각적 확인뿐 아니라 디버그/검증 자동화에 활용되어야 한다.

## 2) 고정 가정 (완화 금지)
- **2-shell 가정 유지**: 입력은 기본적으로 2개의 shell(외피/내피 또는 상/하 구획)로 해석 가능한 케이스를 우선 지원한다.
- **contour 기반 region 구성 유지**: 절단 결과의 region은 삼각형 flood-fill 단독이 아니라, 절단에서 얻은 contour를 기준으로 구성한다.

## 3) 산출물 관점
- 절단 이후 데이터는 재절단(re-slicing)과 디버그 렌더링에 재사용 가능해야 한다.
- 수치 오차를 고려해 epsilon 정책을 일관되게 적용한다.

## 4) 비목표
- 임의의 비다양체(non-manifold) 메시를 완전 자동 복구하는 일반 해석기는 본 범위 밖이다.
- 고품질 오프라인 리메시/불리언 엔진 대체는 목표가 아니다.

## 5) 문서 간 계약
- `prototype_scope.md`는 본 고정 가정을 전제로 MVP/Deferred를 분리한다.
- `technical_design.md`는 2-shell + contour-region 전제를 깨지 않는 데이터 모델/루프를 제시한다.
- `validation_cases.md`는 실패 케이스를 포함해 가정 유지 여부를 검증한다.
- `viewer_spec.md`/`runbook.md`는 개발자 운용 관점의 확인 절차를 제공한다.
