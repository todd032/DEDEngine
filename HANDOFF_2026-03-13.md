# Cookie On The Roof X Handoff

Date: 2026-03-13

## What This Project Is

`Cookie On The Roof X` is a C++20 roof-cookie placement simulation built around SDL2.

Technical direction:

- Windows target through desktop OpenGL 3.3 core
- Web target through Emscripten + WebAssembly + WebGL2
- Android target through Gradle + SDLActivity + GLES 3.0
- The old WPF prototype remains separate and is only a behavior reference

## Current Status

Implemented:

- Roof families: `Flat`, `Gable`, `Dome`, `Pyramid`, `Wave`, `Shed`
- Cookie placement with dense partition-based coverage
- Gable ridge exclusion so cookies are placed on sloped faces only
- Orbit camera with `10deg ~ 60deg` pitch clamp
- Shared SDL input path for mouse and touch
- Browser-compatible SDL lifecycle split into initialize / tick / shutdown
- Emscripten main loop path with WebGL2-targeted output
- GitHub Pages deployment workflow
- Codex Cloud web-build trial scripts
- Headless simulation tests for native builds

Verified on this machine:

- Repo structure is ready for Windows, web, and Android targets

Not yet fully verified on this machine:

- Windows rebuild after the web refactor, because local CMake is missing
- Web build, because local CMake/Ninja/Emscripten were not installed before this handoff
- Android APK build, because Android SDK / NDK / JDK were not installed here

## Build Assumptions

Windows:

- Visual Studio 2022 with Desktop C++ workload
- Windows SDK installed
- CMake available in `PATH` or under `.external-cache/cmake/...`

Web:

- `git`
- `python`
- `cmake`
- `ninja`
- repo-local `emsdk` bootstrapped into `.external-cache/emsdk`

Android:

- Android Studio
- Android SDK
- Android NDK
- JDK 17

## Recommended First Check On A New Machine

1. Open this project folder.
2. Read `README.md` and `FRESH_MACHINE_SETUP.md`.
3. Run `tools/build_web.ps1` or `bash tools/build_web.sh`.
4. Serve `build/web/site` from a static server and validate input + rendering.
5. Enable GitHub Pages to use **GitHub Actions** as the source.
6. After the web path is healthy, return to Windows and Android validation.
